# docs/skills/ — スキル文書庫(AIに参照させるコツ集)

Status: 文書はACTIVE(内容は今から有効な参照知識)。**自動注入の仕組みは未実装**(設計は下記)。

## コンセプト

アシスタントの腕前は「モデルの地力」だけでなく「その場でどんなコツを知っているか」で決まる。ここには**タスク種別ごとの職人知識**を、モデルに直接読ませる形式で置く。マスターが読んでも役立つが、第一の読者はAI。

- 書式: 命令形・具体例つき・1文書60行以内目安(プロンプトに注入されるためトークンは有限)
- 憶測で書かない。モデル固有の作法は出典つきで(image-generation.md 参照)
- 古くなったら直す。「DALL-E時代のコツでGPT Image 2を使う」ような陳腐化がこの文書庫の敵

## 参照のさせ方(設計 — 4o-solo A-2/A-3 の応用)

1. **MVP(今すぐ可能):** チャットで該当タスクを頼むとき、マスターが「skills/recipes.md を参考に」と言う/開発者指示に貼る
2. **次段階:** `server/openai.mjs` の instructions 合成に「タスク種別検出→該当スキル文書を注入」を追加(キーワードヒューリスティックで十分。画像生成ツール呼び出し時は image-generation.md を必ず注入)
3. **将来:** オーケストレータ(4o-thinking設計)がスキル選択も判断

### 優先度の基準

- ★★★ = 頻度が高い＋コツなしだと明らかに品質が落ちる＋このアプリの核機能と関連
- ★★ = 頻度がそこそこ＋コツがあると目に見えて改善
- ★ = たまに頼まれる＋あれば嬉しいが急がない

### 実装の進め方

1. ★★★ のうちアプリ固有のもの(`diary-writing`, `health-advice`, `scheduling`)を先に書く
2. 汎用★★★(`writing-editing`, `email-drafting`, `summarization`, `emotional-support`)を次に
3. 残りは実際に使いながら「ここ精度低いな」と感じた順で追加
4. 各文書は60行以内・命令形・具体例つき(既存4文書と同じ書式)
