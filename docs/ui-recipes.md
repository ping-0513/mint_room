# UIのコツ — 不細工なAI製UIにしないための実践レシピ

Status: ACTIVE GUIDELINE(UI を作る・変更する全エージェント必読。`AGENTS.md` Rule 4 の実践編)。抽象論ではなく、このリポジトリの実コード(`public/styles.css`)を教材にした具体的なコツ集。関連: `docs/master-preferences.md` §1–2(要望)、`docs/ux-design-testing-principles.md`(検証手順)。

## 0. 一番大事な一言

**きれいなUIは「決定の数が少ない」。** 不細工なAI製UIは、コンポーネントごとに毎回新しい色・角丸・影・余白を発明する。きれいなUIは数個の決定を全体に一貫適用しているだけ。センスの差ではなく、規律の差。

## 1. トークンから書く(色を直書きしない)

すべての色・影は CSS 変数から引く。このリポジトリの実例(`styles.css` 冒頭):

```css
:root {
  --mint: #57c9b1;        /* ブランド色。使う場面を絞る */
  --bg / --surface / --surface-2   /* 3層の深度 */
  --text / --text-soft             /* 本文と補足の2段 */
  --border: #d7efe8;               /* 柔らかい境界線 */
  --shadow: 0 4px 18px rgba(87,201,177,0.12);  /* 影は1種類だけ */
}
```

- **コンポーネント内に生のhex値を書いたら負け。** 必要な色がなければトークンを足す(足す前に本当に必要か疑う)。
- グレーを増やさない。「テキスト2段+境界1色」で足りる。AI製UIが不細工になる典型は「微妙に違うグレーが10種類」。

## 2. 決定は1回だけ(半径・影・余白のスケール)

- 角丸は2系統のみ: **ピル(999px)= ボタン・チップ・バッジ**、**10〜16px = カード・入力欄**。3つ目を発明しない。
- 影は `--shadow` の1種類。強調したいからと影を濃くしない(強調は色と余白でやる)。
- 余白は 4/6/8/10/12/16px の階段から選ぶ。19px とか 25px を書き始めたら乱れのサイン。

## 3. 線ではなく面で区切る(3層深度)

- 区切りたいとき、まず `--bg` → `--surface` → `--surface-2` の背景差で区切る。境界線は最後の手段で、使うときも `--border`(低コントラスト)のみ。
- **黒い線・純白・純黒は使わない。** ダーク面も #000 ではなく `#14201d`(このアプリの実値)。まぶしさ・きつさの大半は極端なコントラストから来る。

## 4. ブランド色は出し惜しみする

- `--mint` が出ていい場所: ブランド名、プライマリボタン、アクティブ状態、フォーカスリング、ほんの少しのアクセント(日記の日付など)。**それ以外は無彩色系で我慢。**
- 意味色は danger 1色だけ追加。赤・緑・青・橙が同時に出てくる画面は信号機であってUIではない。
- 「かわいくしよう」として色を足すほど不細工になる。かわいさは色数ではなく、丸み・余白・言葉・小さなきらめきで出す。

## 5. 状態をデザインする(AI製UIが一番サボる所)

ボタンを1個作ったら5状態がセット(`ux-design-testing-principles.md` §2 と同じ):

- hover: 控えめな反応(実例: `border-color: var(--mint)` + 弱いグロー)
- focus: `outline: 2px solid var(--mint-glow)` — キーボード操作者を見捨てない
- disabled: `opacity: .5` + **なぜ押せないかを title で説明**(無言のグレーは禁止)
- 処理中: ボタン無効化+「✨ writing…」のような言葉つき表示
- **空状態: 一番人柄が出る場所。** 実例: 「今日は特にないよ。静かな日🌿」。空をエラー扱いしない

## 6. 動きは「署名」を1つだけ

- このアプリの動きは ✨ の twinkle(1.2秒の opacity 揺れ)**だけ**。位置が動くアニメーションはゼロ。
- 新しいアニメーションを足したくなったら、まず既存の1つで表現できないか考える。entrance連発・バウンス・パララックスは全部禁止(Sol系の悪癖として文書化済み)。
- `prefers-reduced-motion` を必ず尊重。

## 7. タイポグラフィは3サイズ・太字は600まで

- 見出し 1.0〜1.15rem / 本文 1rem / 補足 0.8rem(`--text-soft`)。これで全部。
- **強調は bold 乱発ではなく「補足を薄くする」ことで作る。** 目立たせたい所を増やすのではなく、引っ込める所を増やす。
- line-height 1.5 以上。日本語は詰まると一気に読みにくくなる。

## 8. 絵文字はアイコン、装飾ではない

- 実例: タブ(💬 Chat / 📔 Diary)、確度バッジ(🟢 公式)。**1ラベルに1絵文字+必ずテキスト併記。**
- 見出しにも本文にも絵文字を撒き始めたらコールアウトスパムと同じ道。「意味を運ばない絵文字は消す」。

## 9. 細部の職人技(コピペ推奨の小技)

- チャット吹き出しの片角だけ小さくする(`border-bottom-right-radius: 4px`)— 発言者の方向を向く。既製ライブラリ感が消える定番の一手。
- バッジは小さなピル(`font-size: .7rem; padding: 1px 8px`)で情報を静かに添える。
- カードのグリッドは `repeat(auto-fit, minmax(280px, 1fr))` — 画面幅の変化に黙って追従する。
- 本文の最大幅を制限(このアプリは `max-width: 980px`)。全幅に伸びた文章は読めない。

## 10. コピー(文言)もUI

- マイクロコピーは短く・温かく・命令しない。「Error!」ではなく「うまくいかなかったみたい。もう一度ためす?」。
- ヒントは `hint` クラスの1段落が標準形。警告ボックス・コールアウトを乱発しない。
- ユーザーを責める文言は絶対に書かない(空・失敗・久しぶりの起動、全部)。

## 11. 作業手順のコツ(プロセス編)

1. **新規発明の前に、既存の一番近いコンポーネントをコピーして変える。** このリポジトリなら card / chip / badge / btn / hint が部品箱。
2. 書き終えたら**目を細めて見る**(squint test): ぼやけた状態で階層が読み取れなければ、色か余白の設計が失敗している。
3. **ダークモードと幅375pxで必ず確認**(スモーク台本の項目9)。
4. 最後に「消せる装飾はないか」を1周する。**迷った装飾は消す。** 消して寂しくなったら戻せばいい。

## 12. 不細工カタログ(見つけたら直す)

- グラデーション背景+ガラスモーフィズム+影3種の「テンプレ豪華」
- コンポーネントごとに違う角丸・違う余白・違う青
- 意味のない絵文字の見出し乱発、コールアウトボックス連打
- 押せるのか押せないのか分からない要素、消えないスピナー
- 全要素センタリング、全幅に伸びる文章、詰まった行間
- ライトモードだけ確認された「まぶしい」画面
- 信号機カラーの同時出演

これらは全部「決定が多すぎる」ことの症状。迷ったら §0 に戻ること: **決定を減らし、既存の決定に従う。**
