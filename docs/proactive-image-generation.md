# そこそこに自分から絵を出す — 自発的画像生成の設計メモ

Status: IDEAS ONLY(実装約束ではない設計メモ)。マスターの要望: 「4oが自分から画像生成して出してくる機能も欲しい。ただそこそこにしてほしさもある(笑)」。関連: `docs/skills/image-generation.md`(生成時に必ず注入するコツ)、`docs/4o-thinking-orchestration.md`(判断層)。

## 0. 「そこそこ」を仕様にする

自発的な画像は、うまく効けば魔法(会話が急に豊かになる)、外すとうるさい(コストもかかる)。だから**頻度と場面を設定で縛った上で、既定は控えめ**にする。

## 1. そこそこダイヤル(設定)

| 値 | 挙動 |
|---|---|
| off(**既定**) | 自発生成なし。頼まれた時だけ |
| 提案だけ | 「絵にしてみようか?🎨」と聞くだけ。生成はOK後(1日3回まで提案) |
| ひかえめ | 明確に役立つ場面のみ自動生成、**1日1枚まで** |
| ふつう | 下記トリガー全部有効、**1日3枚まで** |

- 既定 off の理由: 画像生成は1枚ごとに課金される有料API自動実行なので、明示オプトイン必須(CLAUDE.md の方針)。キー設定後の初回に1度だけ「提案だけモードにする?」と案内してよい。
- 残り枚数を設定画面に表示(今日 1/3 など)。透明性がそこそこ感を支える。

## 2. 自発生成のトリガー(許可リスト方式)

出してよい場面(ひかえめ=上2つのみ、ふつう=全部):

1. **説明が視覚的な時** — 配置・構図・見た目の説明(部屋のレイアウト、盛り付け、図解)
2. **日記の挿絵** — その日の日記に小さな1枚(このアプリと相性最高。1日1枚の上限と自然に一致)
3. **お祝い** — 目標達成・継続記録の節目にご褒美カード
4. **レシピの完成イメージ** — 献立提案時の1枚

**出してはいけない場面(常に):** 深刻・つらい話題の最中/ユーザーが急いでいる時/直前に画像を断られた後/ニュースの挿絵(報道画像の代替をAIが作るのは誤解のもと)。

## 3. 挙動の作法

- 生成中も会話を止めない(返答本文を先に、画像は「できたら添える」カード形式)
- 添える一言は控えめに:「描いてみたよ🎨」。自画自賛しない
- 各画像カードに「いいね/いらない」ボタン。「いらない」が2回続いたら自動でダイヤルを1段下げて報告(そこそこ感の自動調整)
- スタイルは `docs/skills/image-generation.md` の mint room スタイルブロックを既定適用(世界観の一貫性)

## 4. アーキテクチャ

- 生成判断: オーケストレータ(または当面は応答後の軽い判定呼び出し)が「トリガー該当+残枚数あり」を確認
- 生成実行: `server/openai.mjs` に `createImage(settings, brief, safetyIdentifier)` を追加(ペイロード集約の原則)。プロンプトは skills/image-generation.md の作法で**サーバー側で組み立てる**
- 画像の保存: MVPはメモリ/一時URLのみ。localStorage に base64 を溜めない(容量を食い潰す)。保存したい画像は明示ダウンロード
- モデルID・サイズ・品質は `MODELS` と同様に設定可能な形で(ハードコードしない)

## 5. 実装ステージ

| Stage | 内容 |
|---|---|
| I1 | 設定にダイヤル(off固定・disabled+説明)と Images タブの説明更新。skills文書(済) |
| I2 | 手動生成: Images タブ+チャットから「描いて」で生成(createImage 実装) |
| I3 | 提案だけモード+日記の挿絵 |
| I4 | ひかえめ/ふつobject+いいね/いらない自動調整 |

各ステージで安全確認: プロバイダのモデレーション準拠、実在人物・画風模倣の拒否、コスト表示。
