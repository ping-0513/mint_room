# docs/tasks/ — タスクチケット置き場

書式とルールは `docs/collaboration-protocol.md` 参照。1チケット=1ファイル=1担当。

## 状態遷移

`TODO` → `IN PROGRESS`(担当が着手時に書き換え) → `REVIEW`(実装完了・報告欄記入済み) → `DONE`(レビュー欄記入済み) / `BLOCKED`(質問欄に理由)

## チケット書式(この形を崩さない)

```markdown
# NNN: タイトル

- Status: TODO
- 担当: Codex(思考最大) / Claude
- 目的: (ユーザー価値を1文で)

## スコープ
- 触ってよいファイル:
- 触ってはいけないもの:

## 受け入れ条件
- (ユーザーに見える挙動の文で列挙)

## 制約
- (従うべきガイドライン・技術制約)

## 検証
- (実行するコマンドとスモーク手順)

## 実装前メモ(担当が着手時に記入。小タスクは省略可)

## 質問(あれば。作業は可能な範囲で続ける)

## 報告(実装者が記入)
- 変更ファイル:
- 検証結果:
- 未確認・懸念:

## レビュー(相手モデルが記入)
```
