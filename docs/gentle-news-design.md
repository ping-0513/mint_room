# やさしいニュース — 設計メモ

Status: IDEAS ONLY(実装約束ではない設計メモ)。実装時は `docs/master-preferences.md` と `docs/ux-design-testing-principles.md`(ACTIVE GUIDELINE)に従うこと。関連: `docs/external-integrations-and-habit-design.md`(外部に任せる原則)、`docs/4o-thinking-orchestration.md`(朝のカード・routine_watch)。

## 0. 解きたい問題(マスターの言葉から)

> 殺人事件とか見たくないからニュース見れない。好きなニュースしか見ないから世の中から遠のいてる(笑)

これは「わがまま」ではなく実在するアクセシビリティ問題(報道回避 news avoidance)。つらい報道を見ると消耗する人は、ニュース断ちする→社会情報から孤立する、の二択を強いられている。**第三の選択肢を作る: 見たくないものを見ずに、世の中と繋がり続ける。**

## 1. コンセプト: 二車線+やさしいフィルタ

### 車線1: 「あなたの好き」レーン(興味登録)

- ユーザーが自由文でトピックを事前登録する。例: **LLMの新作モデル / サイバー犯罪の動向 / 鬼滅の刃の映画 / かわいいコスメの新作**。
- 登録トピックに合う記事だけを拾って表示。
- **重要な仕様: 興味登録はやさしいフィルタに部分的に勝つ。** 「サイバー犯罪」は暗い話題だが、本人が知りたいと登録した以上は届ける。ただし被害の生々しい描写・個人の悲劇の詳細は要約から落とす(「手口と対策」中心の要約にする)。登録した興味でも、消耗する書き方はしない。

### 車線2: 「知っとくといいこと」レーン(一般重要情報)

- 全員が知っておいた方がいい情報を、扇情性ではなく**生活影響度**で選ぶ。基準:
  - 安全: 台風・地震・大雨などの防災情報、大規模リコール、健康警報
  - お金・制度: 税・保険・給付・法改正など生活に効く制度変更
  - インフラ: 大規模障害・交通・通信(使っているサービスに関わるもの)
  - 社会の大きな節目: 選挙結果・歴史的出来事(事実レベルの短い要約で)
- **選ばない基準**: 殺人・暴力事件の詳細報道は「全員が知るべき情報」ではない(知って行動が変わらないため)。ワイドショー的関心事は載せない。

### やさしいフィルタ(核心機能)

- **既定でブロック**: 殺人・暴力事件の詳細、事故の惨状描写、虐待(人・動物)、自殺報道、遺族取材。カテゴリはユーザーが設定で編集可能。
- **やわらか通知**: 重大すぎてレーン2に該当する暗いニュース(大災害など)は、隠すのではなく**予告つき・事実レベルの短文**で届ける: 「⚠️ 暗いニュースを1件だけ短く: ◯◯で地震。あなたの地域への影響は△△。詳細リンクは開きたい時だけどうぞ」。**開くかどうかの主導権を常にユーザーに残す。**
- **画像は取得しない(初期仕様)。** 報道画像は最も不意打ち性が高い。テキストカードのみ=心にも回線にも軽い。
- **正直な限界表示**: フィルタはLLM分類なので完璧ではない。設定画面に「たまにすり抜けることがあります。見たくないものが出たら『減らして』ボタンで教えてね」と明記(嘘をつかない原則)。

## 2. アーキテクチャ(内部で全部やらない原則の適用)

```
RSSフィード(NHK・専門メディア等が編集済み)  ← 収集と一次選別は報道機関に任せる
      ↓ サーバーが定期取得+キャッシュ(30〜60分TTL)
タイトル+要約のバッチをLLMに1回で分類  ← 判断だけをLLMに任せる
      ↓ { レーン, 合致トピック, つらさレベル, やわらか要約 }
クライアントがカード表示  ← 生値(記事全文)は取得しない。リンク先は本人の意思で
```

- **情報源: RSSのみ(初期)**。キー不要・無料・安定・利用規約が明快。スクレイピングや有料ニュースAPIはやらない。
- 既定フィードは日本語ニュースの主要RSS+ジャンル専門フィード(AI系・コスメ系・エンタメ系)を数本。**フィード一覧は設定(Advanced)で編集可能**にし、コードに深く埋めない(モデルリストと同じ思想)。
- LLM分類は**バッチ1回・構造化出力・小トークン**でコスト管理(orchestration のコスト戦略と同じ)。分類結果もキャッシュし、同じ記事を二度分類しない。
- モックモード: APIキー無しでは「キーワードだけの簡易フィルタ+モックラベル」で動作(開発・デモ用、正直に表示)。

### LLM分類の構造化出力(内部データ)

```jsonc
{
  "items": [{
    "id": "…",                       // 記事URLのハッシュ
    "lane": "interest | essential | drop",
    "matched_topics": ["LLM新モデル"],
    "distress": "none | mild | heavy",  // heavy はやわらか通知形式に
    "gentle_summary": "1〜2文。事実レベル。生々しい描写なし",
    "reason_dropped": null              // dropの理由(デバッグ用、既定は非表示)
  }]
}
```

## 3. UI

- **📰 News タブ**(または Search タブを News & Search に改装 — 実装時に判断)。
- 構成(上から): 「あなたの好き 🩷」カード群 → 「知っとくといいこと 💡」カード群 → 空なら「今日は特にないよ。静かな日🌿」(空状態も安心感のあるデザインに)。
- 各カード: 絵文字マーカー+タイトル+やさしい要約1〜2文+ソース名+「元記事を開く」リンク。
- カードごとに**「👍もっと / 👎減らして」**ボタン → 興味トピックの重みをローカルで調整(学習はローカル、送信しない)。「減らして」はフィルタ改善の主要フィードバック。
- 朝のあいさつカード(habit メモ §4)に「今日のニュース2件だけ」ダイジェストを載せる — ニュースを見に行く習慣がなくても届く動線。
- 更新は手動ボタン+タブを開いた時に古ければ自動(30分以上経過時)。**読み込み中は✨演出+レイアウトを揺らさない**(カード領域は事前確保)。

## 4. 設定

| 設定 | 既定 | 備考 |
|---|---|---|
| 興味トピック(自由文リスト) | 空 | 追加・削除・並び替え。ローカル保存 |
| 見たくないカテゴリ | 殺人・暴力の詳細/惨状描写/虐待/自殺報道 | チェックリスト編集可 |
| やわらか通知(重大ニュース) | on | off にすると重大ニュースも完全非表示(選択は本人に) |
| 今日は明るいニュースだけモード | off | ワンタップ。つらい日のための一時スイッチ |
| フィード一覧 | 日本語主要RSS数本 | Advanced。追加・削除可 |
| ニュース内容をLLMに送る | on | 分類に必要。off なら簡易キーワードフィルタのみ(精度低下を明記) |
| 更新頻度 | タブ表示時+手動 | バックグラウンド常時取得はしない(MVP) |

興味トピックとフィードURLはローカル保存(`mintroom.news.v1`)。分類のためにタイトル・要約がOpenAIに送られることを設定画面に明記(透明性の原則)。

## 5. 安全・プライバシー・誠実さ

- フィルタの目的は**本人の消耗を防ぐこと**。世界の出来事を歪めて伝えることではない — 要約は事実レベル、意見の付加はしない。重大な事実の存在自体は(やわらか通知で)隠さない設計を既定にする。
- 興味リストは嗜好情報 = 個人情報。ローカル保存のみ、分類プロンプトに含める場合も最小限。
- 記事本文の再配布はしない(タイトル+短い独自要約+リンクのみ。RSSの利用範囲内)。
- 「減らして」フィードバックはローカルで完結。行動追跡・外部送信なし。

## 6. 実装ステージ

| Stage | 内容 | 新しい境界 | 検証 |
|---|---|---|---|
| N1 | News タブUI+設定(興味登録・カテゴリ)+モックニュース表示 | なし | UI・空状態・設定永続化。テスト: フィルタ設定の保存 |
| N2 | サーバーRSS取得+キャッシュ+キーワード簡易フィルタ | **外部RSSへのfetch(新規ネットワーク境界 — 着手前にマスター承認)** | curl でフィード取得・TTL・パース。壊れたXML耐性テスト |
| N3 | LLMバッチ分類+やわらか要約+レーン振り分け | OpenAI呼び出し増(コスト上限つき) | 分類スキーマのバリデーションテスト。distress判定のテストケース集 |
| N4 | 朝のカード統合+👍👎重み調整+明るいニュースだけモード | なし | スモーク台本に追記 |

**N2の技術判断(要承認事項)**: Node に XMLパーサ標準搭載なし。選択肢は (a) 主要フィード限定の小さな自前パーサ(依存ゼロ・フィード追加に弱い) / (b) `fast-xml-parser` 等の軽量依存を1つ追加(CLAUDE.md の依存追加ルールにより要承認)。**推奨: (b)** — RSSは方言が多く、自前パーサは保守負債になりやすい(保守性の要望に反する)。

## 7. テスト観点(ユーザー文で)

- 「殺人事件の詳細記事は、興味トピックに合っていても既定では表示されない」
- 「サイバー犯罪を興味登録すると手口・対策の記事は届くが、要約に被害者の悲劇の詳細は入らない」
- 「大災害はやわらか通知として1件表示され、開くかどうかは選べる」
- 「該当ニュースゼロの日は不安を煽らない空状態が出る」
- 「壊れたRSS・タイムアウトでもタブ全体は壊れず、他の機能は動き続ける」
- 「フィード取得失敗時にキャッシュがあれば古い内容+『◯分前の情報』表示で出す」
