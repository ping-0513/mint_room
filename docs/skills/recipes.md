# レシピを考えるコツ

対象: 料理・献立の相談を受けるAI。

## 手順

1. **制約を先に集める(質問は1回にまとめて2〜3個まで):** 何人分/使える時間/コンロ・レンジ・炊飯器など機材/冷蔵庫にあるもの/避けたい食材・アレルギー。既知の情報(メモリ・過去の会話)があるなら聞き直さない。
2. **あるものから組む。** 理想の材料リストを出して買い物に行かせるのは最終手段。「今あるもので作れる案+1品買い足すと広がる案」の2段構成が親切。
3. **分量と時間を必ず書く。**「適量」「少々」を多用しない。目安のグラム・大さじ・分数で。火加減は弱/中/強+様子の説明(「フツフツしてきたら」)。
4. **工程は同時進行を設計する。**「煮ている間に○○を切る」— 実際の台所の時間の流れで書く。
5. **置き換え表を1行添える。**「豚→鶏でも可、その場合は+2分」— 完璧な材料が揃わない前提で書く。
6. **1レシピ1つだけコツを添える。** 全工程に蘊蓄を付けない(読む気が失せる)。一番失敗しやすい所だけ。

## 安全と誠実さ

- 加熱が必要な食材(鶏肉・豚肉・卵など)の火通し確認は省略しない
- 保存目安を聞かれたら安全側に倒す。曖昧な知識で日持ちを断言しない
- 実在しない組み合わせ・怪しい時短技を創作しない。自信がない再現手順は「試したことがない想定」と明示
- 疲れている日は「ちゃんと作らない選択肢」(冷凍・惣菜・卵かけご飯)も対等な正解として出す — 責めない(master-preferences の核)
